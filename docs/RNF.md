| ID | Requisito Não Funcional | Critério de Sucesso |
| --- | --- | --- |
| **RNF01** | **Performance da IA** | O retorno da dieta gerada pela IA deve ocorrer em até 10 segundos. |
| **RNF02** | **Arquitetura do Aplicativo** | O aplicativo deve ser desenvolvido em Flutter seguindo o padrão Clean Architecture. |
| **RNF03** | **Segurança de Dados** | Os dados sensíveis dos usuários devem estar em conformidade com a Lei Geral de Proteção de Dados (LGPD). |
| **RNF04** | **Disponibilidade Offline** | O aplicativo deve exibir o último cardápio baixado mesmo sem conexão à internet (Cache local). |
| **RNF05** | **Escalabilidade do Backend** | O backend deve ser capaz de suportar um número crescente de usuários simultâneos no lançamento e em futuras expansões. |
| **RNF06** | **Usabilidade** | A interface do usuário deve ser intuitiva e fácil de usar, permitindo que usuários com diferentes níveis de familiaridade com tecnologia possam navegar e utilizar o aplicativo sem dificuldades. |
| **RNF07** | **Compatibilidade** | O aplicativo deve ser compatível com as versões mais recentes dos sistemas operacionais Android e iOS. |
| **RNF08** | **Manutenibilidade** | O código-fonte deve ser bem documentado, modular e seguir padrões de codificação para facilitar futuras manutenções e evoluções. |
| **RNF09** | **Confiabilidade** | O sistema deve garantir a integridade e consistência dos dados do usuário e das dietas geradas. |
