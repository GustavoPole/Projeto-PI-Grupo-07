| ID | Requisito Funcional | Descrição |
| --- | --- | --- |
| **RF01** | **Gestão de Perfil do Usuário** | O sistema deve permitir que o usuário insira e atualize dados biométricos como idade, peso, altura, sexo e nível de atividade física. |
| **RF02** | **Geração de Dieta Personalizada** | O sistema deve gerar uma dieta semanal personalizada com base nos dados do perfil do usuário e preferências alimentares, utilizando inteligência artificial. |
| **RF03** | **Configuração de Filtros Alimentares** | O usuário deve ser capaz de selecionar e aplicar restrições alimentares (ex: Vegano, Low Carb, Intolerante a Lactose) que serão consideradas na geração da dieta. |
| **RF04** | **Registro de Refeições** | O sistema deve permitir que o usuário marque as refeições sugeridas como concluídas. |
| **RF05** | **Controle de Hidratação** | O sistema deve permitir o registro da ingestão de água e calcular a meta diária de hidratação do usuário. |
| **RF06** | **Acompanhamento de Peso** | O sistema deve gerar um gráfico de evolução de peso com base no histórico de peso inserido pelo usuário. |
| **RF07** | **Cadastro de Usuário** | O sistema deve permitir que novos usuários se cadastrem fornecendo nome, CPF, email e senha. |
| **RF08** | **Autenticação de Usuário** | O sistema deve permitir que usuários existentes façam login utilizando seu email e senha. |
| **RF09** | **Exibição de Mensagens de Feedback** | O sistema deve exibir mensagens de sucesso ou erro para as operações de cadastro e login. |
| **RF10** | **Navegação entre Telas** | O sistema deve permitir a navegação entre as telas de login, cadastro e a tela principal (home) após a autenticação. |
| **RF11** | **Proteção de Rotas** | O sistema backend deve proteger rotas sensíveis, exigindo um token JWT válido para acesso (ex: rota de perfil). |
